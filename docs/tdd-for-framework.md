## Design Doc: TypeScript Bun Starter Platform## 1. Executive Summary
Project Name: TypeScript-Bun-Starter
Objective: Provide a dual-interface platform (CLI & API) for AI agent skills, sharing a single source of truth for business logic and data persistence.
Core Philosophy: Schema-driven, JSON-first, and ultra-low latency.
## 2. Technical Stack

* Runtime: Bun v1.x (Bundler, Test runner, SQLite driver)
* Primary Language: TypeScript (Strict Mode)
* Database: bun:sqlite with Drizzle ORM
* Interfaces:
* CLI: Clipanion (Logic) + Clack (UI/Prompts)
   * API: Hono + @hono/zod-openapi
* Validation: Zod (Single source of truth for DB/API/CLI)
* Tooling: Biome (Lint/Format), LogTape (Logging)

## 3.2. Data Flow

   1. Shared Core: Defines Drizzle schemas and exported Service classes (e.g., MemoryService, ToolService).
   2. CLI Entry: Clipanion parses flags $\rightarrow$ Instantiates Core Service $\rightarrow$ Returns JSON or Clack UI.
   3. API Entry: Hono receives Request $\rightarrow$ Validates via Zod-OpenAPI $\rightarrow$ Calls Core Service $\rightarrow$ Returns JSON.

## 4. Implementation Details## 4.1. Database Strategy

* Schema Inference: Use drizzle-zod to export schemas. This allows the Agent (via API or CLI) to know the exact data structure without manual documentation.
* Migrations: Use drizzle-kit push:sqlite for rapid development cycles common in agent prototyping.

## 4.2. CLI Design (The "Skill" Interface)

* Command Pattern: Every skill is a Command class.
* Agent Mode: Every command must support a --json flag to bypass Clack UI components, allowing agents to pipe output directly.

## 4.3. API Design (The "Remote Tool" Interface)

* Self-Documenting: The /doc endpoint will serve a swagger.json generated by @hono/zod-openapi.
* Auth: Simple API Key middleware for local/private agent use.

## 5. Directory Blueprint

| Path | Purpose |
|---|---|
| packages/core/src/db/schema.ts | Table definitions and Zod exports. |
| packages/core/src/services/ | Logic for skills (e.g., web-search.ts, db-query.ts). |
| apps/cli/src/commands/ | Clipanion command classes. |
| apps/server/src/routes/ | OpenAPI route definitions. |
| biome.json | Linter/Formatter rules (replaces ESLint/Prettier). |

## 6. Success Metrics for Implementation

* Startup Time: CLI execution under 50ms.
* Binary Size: Compiled standalone CLI under 90MB (including Bun runtime).
* Type Coverage: 100% (No any types allowed).
* Agent Compatibility: The API openapi.json must be valid when uploaded to OpenAI/Claude "Actions".
